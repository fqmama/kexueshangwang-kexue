<p>PureVPN<strong>允许种子下载并且不会限制任何类型的流量</strong>。因此，如果你正在寻找种子下载服务，这绝对是一种不错的选择。在我们的测试中，速度非常好。我得到的最高值是<strong>17MB/s</strong>。速度取决于许多因素，因此你的测试结果可能会有所不同。</p>
<h2 class="wp-block-heading" id="purevpn-的功能特点"><span id="PureVPN-9">PureVPN 的功能特点</span></h2>
<h3 class="wp-block-heading" id="stealth-guard">Stealth Guard</h3>
<p>Stealth Guard 是一个独特的PureVPN 功能，它<strong>结合了终止开关和拆分隧道</strong>。你可以选择应用程序或整个网络连接，并选择哪些应用程序只能在启用 VPN 的情况下使用。因此，它超越了拆分隧道，当你的设备和 VPN 服务器之间的连接中断时，选定的应用程序也将被关闭。</p>
<p>假设你正在下载种子，并且你的 VPN 连接断开。如果你将种子下载应用程序添加到 Stealth Guard，不仅你的网络连接会关闭，种子下载客户端也会关闭。</p>
<h3 class="wp-block-heading" id="socks5">SOCKS5</h3>
<p>与其他 VPN 服务提供商不同，<strong>SOCKS5 代理与 VPN 连接密不可分</strong>。因此，要使用它，你必须通过 VPN 连接。它确实限制了潜在的使用场景，但也确实解决了固有的 SOCKS5 代理问题之一。</p>
<p>通常，SOCKS5 不添加任何加密——它只是通过中间 VPN 服务器路由你的连接。但是，添加使用SOCKS5会加强连接的安全性。</p>
<p>另外，连接到服务器时不需要提供身份验证，因为它已经在 VPN 连接期间建立。</p>
<h3 class="wp-block-heading" id="ipv6-网络">IPv6 网络</h3>
<p>PureVPN 已经兼容 IPv6。当你连接到PureVPN 服务器时，<strong>它们都具有IPv4 和 IPv6兼容性</strong>。请注意，VPN 连接本身依赖于额外加密的 IPv4。但是，当你连接到 VPN 服务器时，你可以访问 IPv6 地址，并且所有传出连接的下载速度可能更高。</p>
<h3 class="wp-block-heading" id="vpn多跳">VPN多跳</h3>
<p>如果你正在寻找多跳或双 VPN 功能（通过 2 个或更多 VPN 服务器而不是 1 个服务器连接），那么PureVPN就是你想要的。不过，与大多数竞争对手不同，PureVPN 并未将其包含在应用程序中。因此，设置不会像你习惯的那样容易。</p>
<p>你需要到应用程序中连接到服务器来设置一个连接。然后，根据你的设备，<strong>你必须添加手动连接</strong>。所</p>
<h3 class="wp-block-heading" id="转发端口">转发端口</h3>
<p>端口转发可能是一项有风险的功能，因此PureVPN 开发人员选择了更安全的<strong>通用即插即用</strong>，而不是静态端口转发。更好的是，它仅在应用程序请求时映射网络端口，而不是始终保持打开状态。此外，它不会请求特定的 IP 地址，如果你使用 VPN 匿名，这可能会适得其反。</p>
<h3 class="wp-block-heading" id="固定ip">固定IP</h3>
<p>这里的固定 IP 并不是指静态 IP 地址，它通常单独出售并为你提供自己的唯一 IP 地址。相反，固定 IP 地址类似于<strong>公共 VPN 服务器上的 IP 预留</strong>。</p>
<p>因此，每次你连接到同一个 VPN 服务器时，你都会获得相同的 IP 地址。但如果服务器负载较重，IP 地址也可以分配给其他人。</p>
<p>但是，请记住，该功能不适用于网络托管，因为它们限制了一些端口。</p>
<h2 class="wp-block-heading" id="purevpn-安全吗"><span id="PureVPN-10">PureVPN 安全吗？</span></h2>
<p>PureVPN 是一项安全的 VPN 服务。他们使用的加密标准与市场领导者使用的标准相当。此外，他们的隐私和透明度政策将用户的匿名性置于一切之上。</p>
<h3 class="wp-block-heading" id="加密">加密</h3>
<p>正如我之前提到的，当你连接到PureVPN 服务器时，你的连接是加密的。如果有人正在始监视你的网络，他们将看到的只是被加密的内容。PureVPN 使用<strong>AES-256-CBC</strong>密码来确保是这种安全性。许多安全 VPN使用相同的方法来保护连接，因此该服务不是特例。</p>
<p>进行身份验证时，你的登录名和密码也会被打乱，就像你的连接一样。为此，该服务使用<strong>SHA-256 密码</strong>。因此，他们的服务器仅存储你的凭据的散列版本，这是不可能进行逆向工程的。</p>
<p>尽管这是 VPN 提供商之间的标准加密，但它之所以成为标准是有原因的。浏览网页时，它应该可以确保你的安全。</p>
<h3 class="wp-block-heading" id="vpn协议">VPN协议</h3>
<p>通常，加密不是你在使用 VPN 服务时可以修改的东西。但是，你可以选择VPN协议。这些可能会带来某些优势，在某些情况下可能非常有用。</p>
<p><strong>WireGuard</strong> – 最新和发展最快的VPN协议之一。它使用的代码行数显着减少，并提供了极快的速度。</p>
<p><strong>IKEv2</strong> – 一种较旧的协议，由于其广泛的兼容性而仍然被广泛使用。它还可以在不中断连接的情况下轻松地在 wifi 和移动网络之间切换，非常适合手机、平板电脑和其他便携式设备。</p>
<p><strong>OpenVPN</strong> <strong>(TCP/UDP)</strong> – 一种开源 VPN VPN协议，至今仍是最安全、最稳定的协议之一，尤其是在 TCP 模式下。虽然 UDP 确实牺牲了一些完整性检查周期以使其更快一些，但它因此更容易出错。</p>
<p><strong>SoftEther</strong> – 从技术上讲，这本身并不是一个VPN协议。最好将其描述为使用 SSL over HTTPS 进行数据传输的任何防火墙中打孔的方法。它在限制性制度或严格的住宅网关防火墙后面最有用。不过，它是加密方面最不强大的选项。</p>
<p><strong>SSTP</strong> – 由 Microsoft 制造，此VPN协议是 Windows 独有的。它曾经因其绕过防火墙的能力而受到称赞，也是翻墙科学上网不错的VPN软件协议。</p>
<p><strong>Bolt</strong> – PureVPN上最新的VPN协议。非常安全，并且提供了令人难以置信的速度，可以在互联网审查的国家使用。</p>
<h3 class="wp-block-heading" id="自我管理的-dns">自我管理的 DNS</h3>
<p>依赖第三方 DNS 服务器意味着他们可以访问你的连接日志。因此，如果你使用网络服务提供商分配的 DNS 地址，则可以轻松检索每个访问的域。</p>
<p>PureVPN 使用他们<strong>专有的 DNS 服务器</strong>。因此，当你连接到他们的网络时，你的 DNS 查询将受到与你的 VPN 连接相同级别的加密保护。</p>
<h3 class="wp-block-heading" id="终止开关">终止开关</h3>
<p>大多数 VPN 服务都会宣传，如果你与 VPN 服务器的链接中断，它们的终止开关将关闭你的连接。我决定通过阻止对服务器 IP 的访问并强制关闭应用程序进程来测试它，以测试客户端的反应。</p>
<p>第一次测试产生了好坏参半的结果。在路由器上阻止 IP 地址后，ping 命令仍在运行，这意味着并非所有内容都被阻止。工作中的终止开关会显示无法访问或超时错误，而这里的情况并非如此。DNS 功能被完全禁用，尽管如此，它并没有涵盖所有可能在你的连接断开时可能被利用的漏洞。</p>
<p>第二次测试也进行了类似的测试，在终止进程和启动终止开关之间也有一个短暂的窗口。这仍然留下了一个小的风险窗口，这不是最佳的。</p>
<p>两项测试都表明PureVPN的终止<strong>开关不是防故障</strong>的，因此如果你将 VPN 用于高敏感度任务，则应注意。可能存在终止开关不足的情况。</p>
<h3 class="wp-block-heading" id="日志记录">日志记录</h3>
<p>PureVPN 位于英属维京群岛。它不仅对于 VPN 服务提供商来说是一个有点异国情调的地方，而且对于<strong>五眼联盟</strong>来说也是遥不可及的。英属维京群岛没有数据保留法，因此 VPN 服务提供商不需要存储它们。这允许真正的匿名服务。</p>
<p>根据PureVPN的隐私政策，该服务就是这样 – 完全匿名，并且不保留任何日志。该政策规定他们不保留任何类型的连接信息或交换内容。因此，无论你打算在网上做什么，都不会留下痕迹。</p>
